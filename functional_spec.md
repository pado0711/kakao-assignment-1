## 1. 기술 스택

- **기술 스택:** HTML, CSS, Vanilla JS
- **데이터 저장:** 로컬 스토리지 (JSON 형태)
- **코드 스타일:** Airbnb JavaScript Style Guide

> **📌 참고**
> KST(한국 표준시) 기준 날짜 처리를 위해 `Intl.DateTimeFormat("ko-KR", { timeZone: "Asia/Seoul" })`을 사용한다.

---

## 2. Todo 객체 구조

각 Todo는 아래 필드를 가진 JSON 객체로 저장된다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `id` | string | 고유 식별자 (UUID) |
| `content` | string | 할 일 내용 (필수, 최대 글자 수 지정 필요) |
| `date` | string | 할 일이 속한 날짜 (YYYY-MM-DD, KST 기준) |
| `createdAt` | timestamp | 생성 일시 |
| `updatedAt` | timestamp | 최근 수정 일시 |
| `state` | string | `'전체'` \| `'진행중'` \| `'완료'` |

---

## 3. 기능 명세

### 3-1. Todo 생성

#### 입력 조건

- 할 일 내용(`content`)은 필수 입력 항목이다.
- `content`의 최대 글자 수 제한은 50자이다.
- 달력 입력창에서 날짜를 선택하고 Todo를 생성하면, 해당 날짜의 `date`를 가진 Todo가 생성된다.

#### 초기 State 결정 로직

> **📌 규칙**
> - 생성 시점에서 선택한 날짜 = 오늘(KST) → 초기 state: **`진행중`**
> - 생성 시점에서 선택한 날짜 ≠ 오늘(KST) → 초기 state: **`전체`**

#### 유효성 검사 및 실패 처리

- 입력 값이 비어있으면 추가 버튼을 눌러도 Todo가 생성되지 않는다.
- 생성 실패 시 modal 창을 띄워 실패 알림을 표시한다.
- text 입력창을 강조하여 사용자의 재입력을 유도한다.

---

### 3-2. Todo 목록 정렬

#### 기본 정렬 기준 (날짜별 뷰 내)

- 동일한 날짜 뷰 안에서는 가장 최근에 생성한 Todo가 상위에 오도록 정렬한다.

#### 전체 탭에서의 정렬 기준

- 상태 우선순위: `진행중` → `전체(default)` → `완료`
- 동일 상태 내에서는 가장 최근 생성 Todo가 상위에 온다.

---

### 3-3. Todo 수정

- 각 할 일 항목의 오른쪽 수정 버튼을 클릭한다.
- 기존 텍스트 표시 영역이 수정 가능한 텍스트 입력창으로 전환된다.
- 입력창에는 기존 텍스트가 채워진 상태로 시작하며, 사용자가 내용을 수정한다.
- 엔터 키 입력 시 수정 사항이 저장(update)된다.
- 수정 저장 시 `updatedAt`(수정 일시 timestamp)이 갱신된다.

> **⚠️ 주의**
> 수정 시에도 빈 값 저장을 허용하지 않는다.
> 빈 문자열로 엔터 입력 시 수정이 적용되지 않으며, 생성 실패와 동일하게 입력창 강조 처리를 한다.

- Todo의 날짜가 현재 한국 날짜(KST)와 같다면 state를 `진행중`으로 변경한다.
- `진행중`으로 상태가 변경된 Todo는 리스트의 맨 상위로 이동된다.

---

### 3-4. Todo 완료 처리

- 각 할 일 항목 오른쪽의 체크박스를 클릭하면 해당 내용에 취소선(strike-through)이 적용된다.
- state가 `완료`로 업데이트되고, 리스트 최하위로 정렬된다.

#### 완료 상태 되돌리기

- 완료된 항목의 체크박스를 다시 클릭(해제)하면 취소선이 제거된다.
- state가 `완료`에서 `진행중`으로 되돌아간다.
- 되돌린 Todo는 정렬 기준에 따라 진행중 항목 최상위로 이동된다.

---

### 3-5. Todo 삭제

- 각 할 일 항목에 삭제 버튼을 제공한다.
- 삭제 버튼 클릭 시 해당 Todo를 목록에서 제거하고 로컬스토리지에서도 삭제한다.
- 삭제는 되돌릴 수 없으므로, 실수 방지를 위한 확인 처리(`confirm` 등)를 고려한다.

---

### 3-6. 상태별 탭 보기

> **📌 핵심 원칙**
> 날짜별 뷰와 상태별 탭 뷰는 **독립적으로 동작**한다.
> 상태 탭은 항상 전체 데이터를 대상으로 필터링하며, 현재 선택된 날짜와 무관하게 적용된다.

- **`전체` 탭:** `default` 상태 + `진행중` 상태 + `완료` 상태의 모든 Todo를 조회한다.
- **`진행중` 탭:** 전체 데이터 기준 state = `진행중`인 Todo만 필터링한다.
- **`완료` 탭:** 전체 데이터 기준 state = `완료`인 Todo만 필터링한다.

#### 전체 탭 정렬 기준

- `진행중` → `전체(default)` → `완료` 순으로 정렬한다.
- 동일 상태 내에서는 가장 최근 생성 Todo가 상위에 온다.

#### 예시

> 어제 날짜 선택 후 `진행중` 탭 클릭
> → 날짜 무관, 전체 데이터 중 state = `진행중`인 Todo만 표시된다.

---

### 3-7. 날짜별 뷰

- 화면 상단에 오늘 날짜(KST)가 기본으로 표시된다.
- 이전/다음 버튼으로 날짜를 이동할 수 있다.
- 선택된 날짜에 해당하는 Todo만 표시된다.
- 서로 다른 날짜의 Todo는 각각 독립적으로 관리된다.

---

### 3-8. Pagination

| 항목 | 규격 |
|------|------|
| 페이지당 항목 수 | 6개 |
| 페이지 이동 방식 | 번호 버튼 (1, 2, 3 …) |
| 페이지 초기화 조건 | 탭 전환 또는 날짜 이동 시 1페이지로 초기화 |

---

### 3-9. 로컬스토리지 저장

- Todo의 모든 변경사항(생성, 수정, 완료, 삭제)은 항상 로컬스토리지에 반영된다.
- 새로고침 후에도 기존 데이터가 유지된다.
- 데이터는 JSON 형태로 저장하고 불러온다.
- 로컬스토리지 쓰기 실패(`QuotaExceededError` 등) 시 사용자에게 오류를 알린다.

> **⛔ 한계**
> 로컬스토리지는 브라우저별 약 5MB 한도가 있으며,
> 다른 기기 또는 시크릿 모드에서는 데이터가 유지되지 않는다.
