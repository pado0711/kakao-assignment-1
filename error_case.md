## 1. 개요 및 심각도 기준

본 문서는 Todo 웹 서비스의 기능별로 발생 가능한 모든 에러 케이스를 정의한다.
각 에러는 발생 원인, 사용자 동작, 처리 방법을 포함하며, 구현 시 반드시 핸들링되어야 한다.

| 등급 | 심각도 | 기준 |
|------|--------|------|
| 🔴 | Critical | 데이터 손실 또는 서비스 전체 불가 |
| 🟠 | High | 핵심 기능 사용 불가, 즉시 처리 필요 |
| 🟡 | Medium | 기능 일부 저하, 회피 방법 존재 |
| 🔵 | Low | 사용성 저하, 기능 동작은 가능 |

## 2. 에러 ID 체계

에러 ID는 `[기능 접두어]-[3자리 순번]` 형식을 따른다.

| 접두어 | 기능 영역 | 예시 |
|--------|-----------|------|
| `CR` | Todo 생성 (Create) | CR-001: 내용 미입력 상태로 추가 버튼 클릭 |
| `UP` | Todo 수정 (Update) | UP-001: 수정 후 빈 값으로 엔터 입력 |
| `CP` | Todo 완료 (Complete) | CP-001: 완료 처리 후 로컬스토리지 쓰기 실패 |
| `DL` | Todo 삭제 (Delete) | DL-001: 확인 없이 실수로 삭제 |
| `TB` | 상태별 탭 (Tab) | TB-001: 탭 전환 시 페이지 미초기화 |
| `DT` | 날짜별 뷰 (Date) | DT-001: 날짜 이동 시 페이지 미초기화 |
| `PG` | Pagination | PG-001: 마지막 페이지 이후 다음 버튼 클릭 |
| `ST` | 로컬스토리지 (Storage) | ST-001: 로컬스토리지 쓰기 실패 (용량 초과) |
| `MD` | Modal / 알림 (Modal) | MD-001: Modal 표시 중 배경 스크롤 허용 |

## 3. Todo 생성 (CR)

| ID | 에러 상황 | 발생 원인 | 사용자 동작 | 처리 방법 |
|----|-----------|-----------|-------------|-----------|
| CR-001 | 내용 미입력 상태로 추가 버튼 클릭 | `content`가 빈 문자열(공백 포함) | 추가 버튼 클릭 | Todo 생성 차단 / Modal 알림 / 입력창 테두리 강조 |
| CR-002 | 공백만 입력 후 추가 버튼 클릭 | 공백 문자만 존재하여 실질 내용 없음 | 추가 버튼 클릭 | `trim()` 후 빈 값 판정 → CR-001과 동일 처리 |
| CR-003 | 최대 글자 수 초과 입력 | `content` 길이 제한 초과 | 키 입력 중 | 초과 시점에 입력 차단 or 저장 시 차단 + 경고 메시지 표시 |
| CR-004 | 날짜 미선택 상태로 추가 버튼 클릭 | 달력 입력창에서 날짜 미선택 | 추가 버튼 클릭 | 오늘 날짜를 기본값으로 자동 적용하거나 선택 유도 안내 |
| CR-005 | KST 날짜 계산 오류 | 클라이언트 시스템 시간이 KST가 아님 | 생성 시점 자동 처리 | `Intl.DateTimeFormat`으로 KST 강제 변환하여 날짜 결정 |
| CR-006 | 로컬스토리지 쓰기 실패 (생성) | 저장 공간 초과 (`QuotaExceededError`) | 추가 버튼 클릭 | 에러 캐치 후 저장 실패 알림 / UI에 Todo 추가하지 않음 |
| CR-007 | 동시 탭 생성 충돌 | 두 탭에서 동시에 같은 key로 저장 | 두 탭에서 동시 클릭 | 마지막 저장이 이전 것을 덮어씀 — 현재 스택 한계로 명시 |

## 4. Todo 수정 (UP)

| ID | 에러 상황 | 발생 원인 | 사용자 동작 | 처리 방법 |
|----|-----------|-----------|-------------|-----------|
| UP-001 | 수정 후 빈 값으로 엔터 입력 | `content`를 전부 지우고 엔터 | 수정 입력창에서 엔터 | 저장 차단 / 입력창 강조 / 기존 텍스트 유지 |
| UP-002 | 수정 후 공백만 남기고 엔터 | 공백 문자만 남은 상태로 엔터 | 수정 입력창에서 엔터 | `trim()` 후 빈 값 판정 → UP-001과 동일 처리 |
| UP-003 | 수정 중 최대 글자 수 초과 | 수정 내용이 글자 수 제한 초과 | 수정 입력창 키 입력 | 초과 시점 입력 차단 or 저장 시 차단 + 경고 |
| UP-004 | 수정 중 ESC / 외부 클릭 처리 미정의 | ESC 또는 다른 영역 클릭 | 수정 입력창 활성화 중 | 편집 취소 후 원래 텍스트 복원 (명세 추가 권장) |
| UP-005 | 수정 저장 중 로컬스토리지 쓰기 실패 | 저장 공간 초과 | 수정 후 엔터 | 에러 캐치 후 실패 알림 / 화면 상 텍스트 롤백 |
| UP-006 | 수정 완료 후 KST 날짜 재계산 오류 | 클라이언트 시스템 시간 비정상 | 수정 저장 시 자동 처리 | `Intl.DateTimeFormat`으로 KST 강제 적용 |
| UP-007 | `updatedAt` 갱신 실패 | timestamp 생성 중 예외 발생 | 수정 저장 시 자동 처리 | 에러 로깅 / 저장은 계속 진행하되 `updatedAt` 누락 표시 |

## 8. 날짜별 뷰 (DT)

|----|-----------|-----------|-------------|-----------|
| DT-001 | 날짜 이동 시 페이지 미초기화 | 날짜 변경 후 `currentPage` 초기화 누락 | 이전/다음 버튼 클릭 | 날짜 변경 시 `currentPage = 1` 강제 초기화 후 렌더링 |

| DT-003 | 날짜 경계(자정)에서 오늘 날짜 변경 미처리 | 페이지를 오래 열어둔 상태에서 자정 넘김 | 자정 이후 날짜 관련 동작 | `setInterval` 또는 `visibilitychange` 이벤트로 날짜 재산출 |
| DT-004 | 미래 날짜 Todo 생성 허용 여부 미정의 | 달력에서 미래 날짜 선택 가능 | 달력에서 미래 날짜 선택 | 미래 날짜 허용 여부 명세 결정 및 달력 제한 처리 필요 |
| DT-005 | 과거 날짜의 Todo state 판정 오류 | 과거 날짜 Todo를 수정 시 state를 `진행중`으로 잘못 변경 | 과거 날짜 Todo 수정 | 수정 저장 시 해당 Todo의 `date`와 오늘(KST) 비교 후 state 결정 |

## 9. Pagination (PG)

| ID | 에러 상황 | 발생 원인 | 사용자 동작 | 처리 방법 |
|----|-----------|-----------|-------------|-----------|
| PG-001 | 마지막 페이지 이후 다음 버튼 클릭 | `totalPages` 초과 클릭 | 다음 페이지 버튼 클릭 | `currentPage > totalPages` 조건 체크 / 버튼 비활성화 |
| PG-002 | 1페이지에서 이전 버튼 클릭 | `currentPage < 1` 접근 시도 | 이전 페이지 버튼 클릭 | `currentPage < 2` 조건 체크 / 버튼 비활성화 |
| PG-003 | 필터 결과가 0건일 때 페이지 렌더링 | 탭 전환 후 해당 상태 Todo가 없음 | 탭 또는 날짜 전환 | `"해당 조건의 할 일이 없습니다"` 빈 상태 메시지 표시 |
| PG-004 | Todo 삭제 후 현재 페이지가 범위 초과 | 삭제로 인해 `totalPages` 감소 | 삭제 후 자동 처리 | 삭제 후 `currentPage > 새 totalPages`이면 마지막 페이지로 이동 |
| PG-005 | 페이지 번호 렌더링 오류 | `totalPages` 계산 시 `Math.ceil` 누락으로 정수화 실패 | 렌더링 시 자동 처리 | `Math.ceil(totalItems / ITEMS_PER_PAGE)`로 페이지 수 계산 |

## 10. 로컬스토리지 (ST)

| ID | 에러 상황 | 발생 원인 | 사용자 동작 | 처리 방법 |
|----|-----------|-----------|-------------|-----------|
| ST-001 | 로컬스토리지 쓰기 실패 (용량 초과) | 5MB 한도 초과 (`QuotaExceededError`) | 생성/수정/삭제/완료 처리 | `try-catch`로 `QuotaExceededError` 감지 / 사용자에게 용량 초과 알림 |
| ST-002 | 로컬스토리지 데이터 파싱 오류 | JSON이 손상되었거나 형식이 올바르지 않음 | 페이지 로드 시 자동 처리 | `try-catch`로 `JSON.parse` 감싸기 / 파싱 실패 시 빈 배열로 초기화 |
| ST-003 | 로컬스토리지 데이터 구조 버전 불일치 | 이전 버전 데이터가 남아있는 상태에서 구조 변경 | 페이지 로드 시 자동 처리 | 로드 시 데이터 유효성 검사 후 필수 필드 누락 시 기본값 보정 |
| ST-004 | 로컬스토리지 접근 불가 (시크릿 모드 등) | 브라우저 정책으로 로컬스토리지 차단 | 페이지 로드 시 자동 처리 | 접근 가능 여부 사전 체크 / 불가 시 `"저장 불가 모드"` 안내 |
| ST-005 | 멀티 탭 동시 쓰기 충돌 | 두 탭이 동시에 다른 내용으로 저장 | 두 탭에서 동시 조작 | 마지막 저장이 이전 저장 덮어씀 — 현재 기술 스택(Vanilla JS) 한계로 명시 |
| ST-006 | 로컬스토리지 키 충돌 | 다른 앱/서비스가 동일 키 사용 | 페이지 로드 / 저장 시 | 고유한 네임스페이스 키 사용 (예: `todo-app-v2:todos`) |
