# 비기능 요구사항

## 보안

- 비밀번호는 Argon2로 해시하고 평문을 저장하지 않는다.
- 세션 원문은 DB에 저장하지 않고 SHA-256 해시만 저장한다.
- 세션 쿠키는 HttpOnly, SameSite=Lax, 운영 환경 Secure로 설정한다.
- 모든 사용자 데이터 쿼리에 인증된 `user_id` 조건을 적용한다.
- 변경 요청은 동일 출처인지 검사한다.

## 데이터 무결성

- 요청마다 DB 세션을 닫고 관계 삭제에는 외래키 cascade를 적용한다.
- 반복 규칙에 포함되지 않는 날짜의 예외는 규칙 변경 시 제거한다.
- KST 날짜와 ISO 요일 값을 서버에서 최종 검증한다.

## 유지보수와 검증

- 조회는 Server Component, 상호작용은 Client Component로 분리한다.
- FastAPI URL과 인증 토큰을 브라우저에 노출하지 않는다.
- 프런트 lint·test·build와 백엔드 테스트를 CI 완료 조건으로 삼는다.
