# 비기능 요구사항 문서
**Todo 웹 서비스 | v2.0**

- 조건: 추후의 React 도입으로 인한 리팩토링을 고려하여 확장성 있고 clean한 코드를 작성해야함. Vanila JS로 코드를 작성하되,입력값의 제한 사항을 지키도록 작성할것.

- 체크리스트 : 
    1. 리팩토링하기 좋은 코드란 **"변경에 안전한 코드"**입니다. 

    수정 시 사이드 이펙트가 적고, 의도가 명확하며, 의존성이 낮아 구조 개선 작업이 수월한 코드를 의미합니다. 

    이러한 코드가 갖춘 주요 특징은 다음과 같습니다.

    1. 확실한 테스트 코드 (Safety Net): 변경 사항이 기존 시스템에 영향을 주지 않았는지 가장 빠르고 정확하게 검증할 수 있는 안전장치입니다. 

    단위 테스트(Unit Test)가 잘 작성된 코드는 안심하고 구조를 뜯어고칠 수 있습니다.

    2. 단일 책임 원칙 (SRP) 준수 :** 하나의 클래스나 함수는 단 하나의 책임만 가집니다. 코드가 잘게 쪼개져 있어 특정 기능만 쉽게 이해하고 교체할 수 있는 상태입니다. 

    3. 낮은 결합도와 높은 응집도 (Loose Coupling):** 모듈 간의 의존성이 낮아 한 부분을 수정해도 다른 곳에 미치는 영향이 적습니다. 반면, 관련 있는 로직은 한곳에 모여 있어 코드 파악이 쉽습니다.

    4. 명확한 이름과 의도:** 변수, 함수, 클래스의 이름만 보고도 역할을 알 수 있습니다. ‘매직 넘버’나 의미 없는 약어 대신 명확한 명명 규칙을 따릅니다.
